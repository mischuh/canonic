The `release_gate` pytest marker (`pyproject.toml`: *"conformance-gate tests: must pass (not skip) before beta invites"*) identifies the test suite that guards canonic's frozen serving contract and cross-surface parity. See [What CI checks](/ci-cd/what-ci-checks#release-gate) for how the `release-gate` CI job enforces this.

## What it actually checks

Tests carrying `pytestmark = pytest.mark.release_gate` live across several files, each guarding a different conformance property:

- **`tests/test_conformance_gate.py`**: the frozen-contract gate. Golden JSON schemas are stored under `tests/snapshots/contract_schema_v1/`; any change to the field set of `SemanticQuery`, `QueryResult`, `CompileOutput`, or the error registry fails these tests unless the snapshot is updated *and* `CONTRACT_SCHEMA` is bumped to match.
- **`tests/test_adapter_parity.py`**: verifies the MCP tools and the direct service-level paths produce byte-identical core payloads, so `canonic --json query` and the MCP `query` tool are provably the same thing wearing two transports.
- **`tests/test_exc.py`**: every entry in the error registry (`canonic/exc.py`) is checked against its expected exit code, so the exit-code contract documented in [Error codes](/reference/error-codes) can't silently drift.
- **`tests/e2e/test_walking_skeleton.py`**: an end-to-end parity test against a **live database** (via testcontainers), asserting the CLI and MCP paths return identical results for the same query against real Postgres.
- **`tests/contracts/test_resolver.py`**, **`tests/compiler/test_pipeline.py`**: resolver and compiler behavior that the frozen contract depends on.

## Why "must pass, never skip"

Most of these tests can only run with a real dependency available; the e2e/adapter-parity tests need Docker (via testcontainers) to spin up a real Postgres. A missing Docker daemon makes pytest **skip** rather than fail those tests, which would otherwise let a CI run report green without ever having proven the guarantee.

The `release-gate` CI job closes that gap explicitly:

```bash
uv run pytest -m release_gate --strict-markers -rs --junitxml=release-gate.xml
if grep -q '<skipped' release-gate.xml; then
  echo "::error::Release-gate test(s) were skipped (e.g. Docker unavailable): gate cannot pass silently."
  exit 1
fi
```

A skip is treated as a hard failure for this marker specifically; the gate is designed so it can never pass silently via a missing dependency in the runner.

## Running it locally

```bash
uv run pytest -m release_gate --strict-markers -rs
```

Requires Docker running locally for the testcontainers-backed tests to execute rather than skip.
